## Defects — Prioritised by Severity

### FAIL-1 (HIGH) — `/order/success` is auth-gated — breaks post-payment Stripe redirect

**File:** `src/proxy.ts` line 17  
**Issue:** The `isAuthRoute` matcher is `/order(.*)`, which covers `/order/success`. After a Stripe payment succeeds, the buyer is redirected to `/order/success` — but if their Clerk session hasn't persisted to this tab (or they're on a new device), they get sent to sign-in instead, losing the confirmation page entirely.  
**Spec says:** `/order/success` should be a public confirmation page. The page code itself has no auth check (`"use client"` only).  
**Fix:** Change the route matcher in `proxy.ts` from `"/order(.*)"` to `"/order"` (exact) or use `"/order/track(.*)"` for the tracking route only. `/order/success` should be public.

```typescript
// proxy.ts — current (broken)
const isAuthRoute = createRouteMatcher(["/order(.*)", "/account(.*)", "/track(.*)"]);

// proxy.ts — fix
const isAuthRoute = createRouteMatcher(["/account(.*)", "/track(.*)"]);
// /order/success is public; individual order tracking (/track/[orderId]) is auth-gated separately
```

### FAIL-2 (HIGH) — Hydration mismatch on homepage from `CountdownTimer`

**File:** `src/components/CountdownTimer.tsx` line 32  
**Issue:** `useState(() => getTimeLeft(targetTimestamp))` calls `Date.now()` at component initialisation. During SSR this runs on the server; during hydration it runs again on the client at a slightly different `Date.now()`, producing a different seconds value — React detects the mismatch and regenerates the tree. This emits a console error and causes a flash/flicker on initial load.  
**Spec says:** No console errors in browser.  
**Fix:** Initialise with a static value and let `useEffect` hydrate the live time:

```typescript
// CountdownTimer.tsx — fix
const [timeLeft, setTimeLeft] = useState({ days: 0, hours: 0, minutes: 0, seconds: 0, totalSeconds: -1 });

useEffect(() => {
  setTimeLeft(getTimeLeft(targetTimestamp)); // hydrate immediately
  const timer = setInterval(() => setTimeLeft(getTimeLeft(targetTimestamp)), 1000);
  return () => clearInterval(timer);
}, [targetTimestamp]);

// Render nothing (or a skeleton) if totalSeconds === -1 (before hydration)
if (timeLeft.totalSeconds === -1) return <span className={`font-mono tabular-nums ${baseColor ?? 'text-ocean-teal'} ${className}`}>—</span>;
```

### WARN-1 (MEDIUM) — Cart "Remove" button is icon-only (no accessible text node)

**File:** `src/components/CartDrawer.tsx`  
**Issue:** The remove button uses `aria-label="Remove Walu"` but has no visible text "Remove". This is fine for accessibility (aria-label is correct) but the QA selector `text=Remove` won't find it. Not a bug — just noting the pattern uses icon-only with aria-label.  
**Action:** No fix needed. WARN downgraded.
