## Known Issues

### #1 — WARN LOW: FJ391 not detected by Playwright text locator at 1280px
- **What:** `page.locator('text=FJ391').textContent()` returns null at 1280px desktop viewport.
- **Why:** The element uses `hidden sm:inline`. At 1280px Tailwind's `sm:` breakpoint (640px+) should apply, making it visible. The locator may be failing due to Playwright's text-matching behaviour with non-ASCII characters (✈️ &nbsp; prefix) or a timing issue in headless mode.
- **Impact:** Low — element is visually present (confirmed in screenshot `03-delivery-banner.png`).
- **Fix:** Replace `hidden sm:inline` with `hidden sm:flex` or use explicit inline-flex in the banner, and update the test to use `page.getByText('FJ391', { exact: false })`.

### #2 — WARN MEDIUM: Walu not in fish grid in April
- **What:** Walu (Spanish Mackerel) is the launch product and the spec says it should be first in the grid. It is absent from the homepage in April because the seed data has `month_start=5` (May).
- **Spec says:** "Walu should be the FIRST card" — this is only achievable if Walu is in season.
- **Root cause:** Seed data sets Walu season May–October. Current month is April.
- **Business decision needed:** Should Walu season start April or May? If April, run:
  ```sql
  UPDATE seasons SET month_start = 4
  WHERE fish_species_id = (SELECT id FROM fish_species WHERE name_fijian = 'Walu');
  ```
- **Impact:** Medium — Walu is the headline product. Without it, Donu (Coral Trout) leads the grid. The sort logic is correct and will put Walu first once it enters season.
- **Note:** The seasonal filter is working correctly — this is a data issue, not a code issue.
